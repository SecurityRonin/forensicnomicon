forensic-catalog — Architecture

Zero-dependency Rust crate · DFIR indicator tables · Structured artifact catalog

Indicator Tables (10 modules)

  • • Zero dependencies — std-only, embeds anywhere
  • • Each module is independent and additive
  • • const-friendly: all data lives in static slices
  • • Every entry cites MITRE ATT&CK + source URLs
  • • Iterator chaining (no slice duplication)

Core Catalog — artifact.rs

  • • 150+ static ArtifactDescriptor entries
  • • Decoder enum handles ROT13, FILETIME, binary layouts
  • • ContainerProfile / RecordSignature for format recognition
  • • ForensicCatalog: query by id, MITRE, triage priority
  • • decode() → ArtifactRecord with fully typed fields

CI/CD & Publishing

  • • CI: fmt · clippy · test (Linux/Win/macOS) · deny
  • • MSRV: Rust 1.75+
  • • Docs auto-published to GitHub Pages on push
  • • Releases triggered by semver tags → crates.io
  • • Daily feed-watch: OPML + RSS source refresh
